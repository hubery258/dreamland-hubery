    收支记录app

    🎯 项目目标（Goals）

    1. 简洁高效的记账体验

    2. 开发一个无广告、轻量化的应用，能够用最少的步骤快速记录每一笔支出。

    3. 学习与成长

    4. 将此作为 CS50 的 Final Project，锻炼编程能力与项目管理（PM）能力，完成一个完整的 CS 项目。

    5. 移动端可用性

    6. 应用需支持手机端使用，最好能以 PWA 形式安装到首页并离线运行。

    7. 个人满足感

    8. 在日常生活中使用自己的作品，获得成就感与持续动力。

    💎 项目价值（Value）

    • 对个人生活：提供一个真正实用的工具，替代临时记账方案，让记账变得轻松自然。
    • 对学习成长：这是你第一个完整的 CS 项目，能让你掌握从需求分析到实现的完整流程。
    • 对未来发展：作为作品集展示，证明你具备独立开发和项目管理的能力；同时为后续扩展（预算提醒、多设备同步等）打下基础。
    • 对心理层面：通过亲手打造并使用自己的应用，获得满足感和自我认同。

    📝 建议

    • 目标和价值区分：目标更偏向“要做什么”，价值更偏向“为什么值得做”。你写的价值和目标确实有重叠，但这是正常的，因为一个好项目的目标本身就能体现价值。
    • 愿景声明（可选）：你可以把目标和价值浓缩成一句话，写在项目首页或 README 里，比如：

      “打造一个简洁、轻量、可在手机端使用的记账应用，既满足日常需求，也作为我的第一个完整 CS 项目，见证学习与成长。”

    完全可以分成 基础功能 和 扩展功能，这样既能保证你先做出一个可用的 MVP（最小可用产品），又能为后续迭代留出空间。

    🪴 基础功能（MVP 必备）

    这些是最小化可用的功能，完成后你就能真正用它来记账： - 添加支出/收入：输入金额、日期、分类、备注。
    - 查看账目列表：按时间顺序显示所有记录。
    - 分类与类型：至少区分收入/支出，支持简单分类（如“餐饮”“交通”）。
    - 统计汇总：查看某一时间段的总支出/收入。
    - 移动端适配：界面在手机上能正常使用。
    - PWA 支持：可以添加到手机首页，离线使用。

    🌱 扩展功能（迭代升级）

    这些功能不是必须，但能让应用更强大、更贴近真实需求： - 账户管理：支持银行卡、现金、支付宝等不同账户。
    - 标签系统：为账目添加标签（如“旅行”“学习”），方便筛选。
    - 预算提醒：设置月度预算，超支时提醒。
    - 数据导入/导出：支持导出到 CSV/Excel，或从 SQLite 导入。
    - 多用户支持：登录系统，支持多设备同步。
    - 可视化报表：用图表展示收支趋势。

    🧭 建议的推进方式

    • 第一阶段：先完成基础功能，保证能用。
    • 第二阶段：逐步加入扩展功能，每次只加一个小功能。
    • 第三阶段：优化体验（UI、交互、性能）。

    这样分层后，你就能避免一开始就陷入复杂功能，而是先做出一个能用的版本，再逐步迭代。
